## Plan Contract Summary

### Description

Story `0000051` adds GitHub Copilot as a third top-level chat provider alongside Codex and LM Studio, with chat-only scope. The work extends provider contracts, runtime selection, auth flow, model listing, streaming, persistence, and Docker or env wiring without broadening into agent, command, or flow execution.

### Acceptance Criteria

The canonical plan requires all of the following:

1. Three-provider contract support (`codex`, `copilot`, `lmstudio`) across server, client, shared types, OpenAPI, defaults, fallback, provider ordering, and deterministic readiness behavior.
2. Copilot provider and model discovery through the shared provider and model shapes, including stable reason precedence and honest partial metadata handling.
3. Copilot chat execution through the existing event bridge, deterministic session continuity, and persisted conversation or turn records using provider `copilot`.
4. Shared auth UX and shared auth contract for Codex and Copilot, with two-phase device flow, Docker-safe browser handoff, persisted auth reuse, and deterministic pre-start failure reporting.
5. Consistent `CODEINFO_COPILOT_HOME` runtime wiring across development, local Docker, and e2e isolation without introducing new ports or source bind mounts.
6. Fake or mocked automated coverage for Copilot, visible unavailable reasons when Copilot is not ready, no reinterpretation of Codex-only flags for Copilot, and no regression to existing Codex or LM Studio behavior.

- Copilot BYOK provider-routing UX

- Copilot permission-control UI
- In-place model switching for an existing transcript
