## Story Snapshot

### Description

Story `0000052` replaces owner-based `target: "current"` and bulk `target: "all"` authored re-ingest targets with:

- `target: "working"` for the selected run `working_folder` repository only.
- `target: "plan_scope"` for the selected working repository plus any `additional_repositories[].path` entries from `<working-repo>/codeInfoStatus/flow-state/current-plan.json`.

The story also requires:

- best-effort ordered batch behavior with warnings instead of hard batch failure for handoff degradation or per-repository failures;
- reused batch transcript payloads instead of inventing a second plan-scope-only shape;
- stable request-scoped repository listing reuse;
- downstream summary ownership staying with the execution layer;
- explicit legacy `all` to canonical `plan_scope` read-side normalization;
- hardened shared review prompts and review flows with a risk-invariant matrix, rejected-risk notes, and a blind-spot challenge step.

## Backward-Compatibility And Planned Edge-Case Deviations

- Legacy stored batch `targetMode: "all"` is intentionally still readable through the lifecycle normalization path, but new writes are canonical `plan_scope` only.
- Missing, malformed, unreadable, or structurally invalid handoff data intentionally degrades to working-only with warnings instead of hard failing the batch. That is a deliberate story contract, not a generic validation omission.
- Skipped-at-resolution repositories intentionally stay out of the attempted `repositories` array and `summary` counts. That is explicitly permitted by the plan even though another design might have synthesized pseudo-results.
- `working` and `plan_scope` remain command/flow-only authored targets; MCP stays explicit-`sourceId` only by design.
