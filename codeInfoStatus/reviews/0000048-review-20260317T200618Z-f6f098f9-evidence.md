## Plan Extraction

### Description

Story 48 combines three coordinated changes:

1. Reference resolution for flows and commands moves from owner-first to working-repo-first, with one shared four-slot repository order: working repo, owner repo, local `codeInfo2`, then other ingested repos.
2. The selected working folder becomes a persisted conversation-level contract across chat, agents, flows, and direct agent-command execution, including restore, edit, invalid-path clearing, and child-conversation inheritance behavior.
3. Repository-owned environment variables are cut over to `CODEINFO_` / `VITE_CODEINFO_`, and OpenAI ingest token counting moves from a heuristic to tokenizer-backed counting with the provider-truth 8192-token limit.

### Acceptance Criteria

The active plan defines 46 acceptance criteria covering:

1. shared working-repo-first repository ordering, nested restart behavior, duplicate handling, fail-fast vs fallback behavior, structured lookup logs, and compact lookup metadata;
2. persisted working-folder behavior across chat, agent, flow, and direct-command surfaces, including restore/edit/lock/clear semantics;
3. repository-wide env cutover to `CODEINFO_` / `VITE_CODEINFO_` with no checked-in legacy repo-owned names left behind;
4. tokenizer-backed OpenAI counting, 8192-token limit truth, deterministic oversized-input classification, and no silent heuristic fallback.

### Out Of Scope

The plan explicitly excludes:

1. new reference types or new free-form path syntax;
2. changing repository asset subfolder conventions;
3. broad flow discovery changes unrelated to surfacing the working-repo signal;
4. weakening fail-fast behavior for invalid higher-priority candidates;
5. renaming non-repo-owned environment variables like `HOME`, `PATH`, or `NODE_ENV`;
6. broader Playwright networking changes;
7. broader ingest-batching redesign beyond the tokenizer-backed OpenAI fix.
