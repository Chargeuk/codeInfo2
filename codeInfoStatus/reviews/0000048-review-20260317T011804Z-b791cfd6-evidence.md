## Scope Extract

### Description

Story 48 bundles four linked changes:

1. Move command and markdown reference resolution to a shared working-repo-first candidate order: working repository, referencing-file owner, local `codeInfo2`, then other ingested repositories.
2. Persist and restore the selected working folder across chat, agent, flow, and direct-command surfaces using existing conversation/turn storage rather than creating new persistence concepts.
3. Rename repo-owned environment variables to `CODEINFO_` and `VITE_CODEINFO_` with one coordinated cutover across code, compose, wrappers, tests, and docs.
4. Replace OpenAI heuristic token counting with tokenizer-backed counting using `tiktoken`/`cl100k_base`, keep the real 8192-token limit, and fail closed instead of silently falling back.

### Acceptance Criteria

Key acceptance themes from the active plan:

- Shared working-repo-first lookup ordering must apply everywhere command and markdown references resolve, including nested hops and direct command execution.
- Working folder state must be saved, restored, edited while idle, locked while running, cleared when stale, and inherited by flow-created child agent conversations.
- Lookup debugging must be visible in structured logs and compact runtime metadata without inflating high-level API responses.
- Env cutover must be a clean rename to `CODEINFO_` and `VITE_CODEINFO_` with no checked-in dual-read leftovers.
- OpenAI counting must switch to tokenizer-backed counting everywhere the heuristic previously affected behavior, with exact 8192-boundary coverage and no heuristic fallback.

### Out Of Scope

The plan explicitly leaves these out:

- changing repository subfolder conventions for command or markdown assets;
- adding new reference types or free-form path syntax;
- changing flow discovery/listing except where needed to surface the working repository signal;
- changing fail-fast policy for invalid higher-priority candidates;
- inventing separate working-repo concepts per transport/surface;
- renaming non-repository-owned env vars like `HOME`, `PATH`, or `NODE_ENV`;
- reworking the Playwright networking model beyond documentation;
- broad ingest-batching redesign beyond the OpenAI counting fix.
