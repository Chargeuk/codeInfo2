## Plan Summary

### Description

Story 48 has three coordinated feature areas plus the review follow-up chain that closed them:

1. Reference resolution moved from owner-first to working-repo-first for flow-command, flow-markdown, command-markdown, and direct-command lookups, with one shared repository-candidate contract, deterministic duplicate handling, fresh restart of the search order on nested lookups, structured lookup logging, and compact run metadata.
2. The selected working folder became a persisted and editable conversation-level concept across chat, agents, flows, and direct agent commands, with invalid-path clearing, restore behavior, idle-edit semantics, and inheritance into child agent conversations created by flows.
3. Repo-owned env variables were cut over to `CODEINFO_` / `VITE_CODEINFO_`, and OpenAI ingest token counting moved from heuristics to tokenizer-backed counting with deterministic oversized-input classification.

### Acceptance Criteria

The plan defines `AC01` through `AC48` covering:

- working-repo-first candidate ordering and nested reference restart behavior;
- persisted working-folder save, restore, validation, invalidation, and lock semantics across all surfaces;
- structured lookup logging and compact lookup metadata;
- full env cutover to `CODEINFO_` / `VITE_CODEINFO_`;
- tokenizer-backed OpenAI counting, 8192-token truth, no silent heuristic fallback, and lifecycle handling.
