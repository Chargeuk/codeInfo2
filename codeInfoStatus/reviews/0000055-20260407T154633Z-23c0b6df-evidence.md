## Review Hotspots For The Findings Pass

- `server/src/ingest/ingestJob.ts`
  - Admission-vs-execution validation gap: findings must compare route-time `validateExecutableIngestInput()` with queue-promotion-time validation and startup-recovery replay.
  - Merge-before-validate ordering: `pumpIngestQueue()` promotes and assigns a real `runId` before the run processor validates execution readiness.
  - Zero-work fast path invariant: when `operation === 'reembed'`, `deltaMode === 'delta'`, and `deltaWorkCount === 0`, the run should be able to finish without embedding-model lookup or Chroma bootstrap being required for success.
  - Waiter lifecycle: `waitForQueueRequestTerminalStatus()` registers an event listener plus timeout fallback and must unregister on terminal, timeout, setup-read failure, and early settle.
- `server/src/ingest/reingestService.ts`
  - Canonical-vs-mounted path split: findings must challenge whether `canonicalTargetPath` remains the bookkeeping identity while `requestPayload.path` remains only the execution path.

- `server/src/lmstudio/toolService.ts`
  - Overlay precedence: findings must challenge whether queue documents always beat stale persisted metadata for live fields, and whether mixed-path active overlays still collapse to one authoritative row.
  - Size-growth note: queue-list queries scale with queued request count, not per-file or per-chunk counts; current story scope is whole-request queueing, so no batching is implemented or required by the plan.
