## Description

Story 0000047 makes Codex chat/runtime config behavior deterministic in one place. The planned outcome is that `codex/chat/config.toml` becomes the effective default model source, the shared Codex capability/default path owns merged model availability, missing `codex/config.toml` and `codex/chat/config.toml` bootstrap from canonical in-code templates instead of checked-in sample files, shared base runtime settings still flow into chat/agent runtime resolution after chat bootstrap becomes minimal, and Context7 auth is normalized in memory via `CODEINFO_CONTEXT7_API_KEY` without rewriting config files.

## Acceptance Criteria

The plan has 36 acceptance criteria covering five themes:

1. shared Codex model-list and default-model resolution across REST and MCP surfaces
2. deterministic bootstrap of missing `codex/config.toml` and `codex/chat/config.toml`
3. preservation of shared base runtime inheritance after direct chat bootstrap
4. in-memory Context7 API-key normalization for local stdio `args` definitions only
5. unchanged public REST/MCP payload shapes

## Out Of Scope

The plan explicitly excludes prompt/tool-guidance changes, workflow-step behavior changes, auth propagation changes, rewrite-on-read config mutation, broad provider-selection changes beyond Story 47 defaults/listing, new hard-error contracts for invalid chat config, silent repair of invalid existing config files, remote/header-based Context7 normalization, generic credential validation, and generic migration-framework work.
