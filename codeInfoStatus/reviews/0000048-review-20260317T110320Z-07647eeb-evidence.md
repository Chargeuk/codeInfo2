## Plan Summary

### Description

Story 48 combines four coordinated changes:

1. Replace owner-first command and markdown reference resolution with a shared working-repo-first order: working repository, owner repository, local `codeInfo2`, then other ingested repositories.
2. Persist and restore the selected working folder across chats, agent conversations, flows, and direct command execution, using the owning conversation record plus per-run snapshots.
3. Perform a clean repository-wide env cutover from generic names to `CODEINFO_` and `VITE_CODEINFO_` names.
4. Replace OpenAI embedding token heuristics with tokenizer-backed counting and keep oversized-input classification aligned with provider truth.

The plan also explicitly requires deterministic duplicate removal, per-hop lookup-order restart for nested references, fail-fast behavior for higher-priority invalid candidates, safe missing-working-folder degraded mode, invalid saved-folder clearing, and observability surfaces that make lookup decisions understandable.

### Acceptance Criteria

The plan defines acceptance criteria for:

- shared working-repo-first lookup order across flow-command, flow-markdown, command-markdown, and direct-command surfaces;
- shared repository-candidate contract, duplicate removal, nested-hop restart, degraded-mode handling, fail-fast validation, structured lookup logs, and compact runtime lookup metadata;
- saved working-folder persistence, restore, editability, invalid-path clearing, child-agent inheritance, and active-run locking across chat, agent, flow, and direct-command UX;
- full server/client env cutover to `CODEINFO_` and `VITE_CODEINFO_` names with no checked-in legacy references left behind;
- tokenizer-backed OpenAI counting, 8192-token provider truth, deterministic oversize classification, and no heuristic fallback once tokenizer-backed counting exists.

### Out Of Scope

- New workflow asset subfolder conventions.
- New reference types or free-form path syntax.
- Flow-file discovery redesign except where needed to expose the chosen working-repository signal.
- Changing the documented fail-fast policy.
- Introducing transport-specific working-repository concepts.
- Renaming non-repository-owned env vars.
- Broader Playwright networking redesign.
- Broader ingest batching redesign beyond the OpenAI token-counting fix.
