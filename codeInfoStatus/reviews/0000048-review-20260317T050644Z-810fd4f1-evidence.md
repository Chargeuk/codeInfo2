## Scope Extract

### Description

Story 48 implements four linked changes:

1. Move command and markdown reference resolution to one shared working-repo-first repository order: working repository, referencing-file owner, local `codeInfo2`, then other ingested repositories.
2. Persist, restore, edit, lock, and invalidate working-folder state across chat, agent, flow, and direct-command surfaces using existing conversation and turn records rather than new persistence concepts.
3. Rename repository-owned environment variables to `CODEINFO_` and `VITE_CODEINFO_` across server, client, compose, wrappers, tests, and docs.
4. Replace OpenAI heuristic token counting with tokenizer-backed counting using `tiktoken`/`cl100k_base`, keep the real 8192-token model limit, and fail closed instead of silently falling back.

The reopened review-fix work also closed four review findings:

5. Make chat enforce the same ingested-repository working-folder contract as the other surfaces.
6. Distinguish stale working folders from operational filesystem failures.
7. Preserve distinct case-sensitive repository roots when de-duping candidate order.
8. Surface malformed canonical client runtime config instead of silently defaulting it away.
9. Repair the flow-loop stop/cancel test cleanup seam so the full server-unit summary wrapper is once again a trustworthy final validation gate.

### Acceptance Criteria

The active plan defines 46 acceptance criteria across five themes:

- shared working-repo-first lookup order for commands and markdown, including nested lookups and direct command execution;
