## Canonical Plan Extract

### Description Summary

- Story 53 introduces `executionId` as the flow-execution boundary so fresh starts get fresh parent and child conversations while resume stays on the same execution.
- Parent flow state remains in `conversation.flags.flow`.
- Flow-created child conversations remain normal agent conversations, visible in Agents, and resume must use the current live child conversation state after manual chat.
- The UI adds a lightweight `Run <shortExecutionId>` metadata clue in existing sidebar metadata for both flow parent rows and flow-created child rows.

### Acceptance Criteria Summary

- Server/runtime: fresh runs mint and persist execution identity, do not leak slot reuse across executions, and preserve same-execution reuse.
- Resume/compatibility: legacy parent and child state backfill must keep resume working and fail clearly on bad child mappings.
- UI: fresh Run creates a new parent conversation, Resume reuses the stopped conversation, and sidebars show `Run <shortExecutionId>` without renaming titles.
- Validation: final wrapper-backed server, client, e2e, compose, and manual Playwright proof are recorded in the completed plan.
