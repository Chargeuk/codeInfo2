## Plan Extract

### Description

Story 49 addresses long-transcript client slowness, especially Agents input lag while a long transcript is visible. The plan centralizes transcript rendering under a shared client layer, applies the shared path across Chat, Agents, and Flows, adds virtualization and explicit scroll or anchor behavior for long conversations, preserves existing hydration and retained-assistant semantics, and permits one narrow server-side exception for deferred-final Flow or coding-agent stop alignment plus stop-path diagnostics.

### Acceptance Criteria

1. Long visible transcripts remain responsive, including Agents typing.
2. Agents typing no longer rerenders the whole rich transcript on each keystroke.
3. Transcript rendering is refactored into reusable shared client pieces.
4. The shared transcript path is applied consistently to Chat, Agents, and Flows.
5. Chat, Agents, and Flows no longer own separate inline main-transcript `messages.map(...)` loops.
6. Shared rendering preserves the existing newest-last visible order.
7. Long transcripts no longer mount and reconcile every row when only a window is needed.
8. Rich transcript features still work, including markdown, status metadata, tools, citations, and thought-process UI where those already existed.
9. Citation rendering remains page-configured, so Flows does not gain citation UI.
10. History hydration and ordinary UI interactions avoid unnecessary whole-transcript replacement.
11. The optimized transcript still consumes the existing message and hydration sources.
12. Virtualization or windowing supports variable-height rows.
13. Virtualization uses stable `message.id` keys, dynamic measurement, and bounded overscan.
14. Bottom-pinned versus scrolled-away behavior stays correct.
15. Row remeasurement after streaming or expand/collapse stays correct.
16. Rich-row expansion state survives virtual unmount or remount.
17. Shared transcript owns transcript rendering while page-specific controls remain outside it.
18. Shared transcript owns the real scroll and anchor contract.
19. Stable identity, hydration/inflight merge behavior, and Flows retained-assistant behavior are preserved.
