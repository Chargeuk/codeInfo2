## Plan Summary

### Description

Story 48 combines four coordinated changes:

- switch flow and command reference resolution from owner-first to working-repo-first using one shared ordered-repository contract;
- persist and restore the selected working folder across chat, agent, flow, and direct-command surfaces, including child flow-created agent conversations;
- complete a clean repository-owned env cutover to `CODEINFO_*` and `VITE_CODEINFO_*`;
- replace OpenAI heuristic token counting with tokenizer-backed counting aligned to provider truth.

The story also makes lookup/debugging observable through shared markers and compact persisted runtime metadata, while keeping fail-fast behavior for higher-priority parse/validation/decode errors and preserving the existing conversation-based storage model.

### Acceptance Criteria

The plan defines 46 acceptance criteria covering:

- shared repository candidate ordering and nested restart behavior;
- working-folder persistence, restore, edit, validation, stale-path clearing, and child-conversation inheritance;
- structured lookup logging and compact persisted lookup summaries;
- clean env-name cutover with no checked-in legacy product-owned names left in active docs/tests/configs;
- tokenizer-backed OpenAI input counting, 8192-token provider-truth enforcement, and no fallback to heuristic or whitespace estimation on tokenizer failure.

### Out Of Scope

The plan explicitly excludes:

- changing workflow-asset on-disk subfolder conventions;
- adding new reference types beyond commands and markdown;
- inventing free-form workflow path syntax or variable substitution;
- weakening fail-fast behavior for unreadable or invalid higher-priority candidates;
- defining separate current-working-repository concepts per transport or UI surface;
- renaming non-repository-owned environment variables such as `HOME`, `PATH`, or `NODE_ENV`;
- broader Playwright container-networking redesign;
- broader ingest-batching redesign beyond the OpenAI token-counting fix.
