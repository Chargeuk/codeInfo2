## Plan Summary

### Description

Story 48 combines four coordinated workstreams:

1. Switch referenced-file lookup from owner-first to working-repo-first with one shared repository-candidate contract.
2. Persist, restore, edit, validate, and clear working-folder state across chat, agent, flow, and direct-command surfaces.
3. Complete a repo-owned environment-variable cutover to `CODEINFO_*` and `VITE_CODEINFO_*`.
4. Replace OpenAI heuristic token counting with tokenizer-backed counting aligned to the provider’s 8192-token truth and fail-closed behavior.

The plan also requires shared lookup/debug markers, compact persisted lookup summaries, fail-fast behavior for higher-priority parse/validation/decode failures, and reuse of the existing conversation-based storage model rather than inventing new record types.

### Acceptance Criteria

The plan defines 46 acceptance criteria spanning:

- shared repository candidate ordering, nested restart behavior, duplicate de-duplication, and command/markdown/direct-command resolution;
- working-folder persistence, restore, idle edit, lock-while-running, stale-path clearing, fail-closed validation, and child-agent inheritance;
- structured lookup logging plus compact persisted runtime lookup summaries;

- tokenizer-backed OpenAI counting, 8192-token hard limit alignment, deterministic input-too-large classification, and no fallback to heuristic/whitespace estimation when tokenizer counting fails.
