# Project Structure

**Update this file whenever files are added/removed/renamed; every task that changes structure must refresh this map.**

Root files/directories (as of initial tooling setup):

- AGENTS.md — working rules for AI agents and workflow expectations.
- README.md — overview, prerequisites, root commands, env policy.
- design.md — design notes; now includes tooling baseline.
- projectStructure.md — this living map.
- package.json — npm workspaces + root scripts + lint-staged/Husky setup.
- package-lock.json — locked deps (generated by `npm install`).
- tsconfig.base.json — shared TypeScript compiler options.
- tsconfig.json — project references to client/server/common.
- eslint.config.js — flat ESLint config.
- .prettierrc / .prettierignore — Prettier settings.
- .editorconfig — whitespace/newline defaults.
- .npmrc — npm config (`save-exact=true`).
- .gitignore — ignores node_modules, dist, coverage, `.env.local`, etc. (keeps `.env` tracked).
- .husky/ — git hooks (pre-commit runs lint-staged).
- planning/ — story plans and templates.
  - planning/plan_format.md — template and instructions for writing story plans.
  - planning/0000001-initial-skeleton-setup.md — plan for initial skeleton story and tasks.
- client/ — React 19 client workspace (scaffold coming in later tasks; currently placeholder package.json).
- server/ — Express server workspace with `package.json`, `tsconfig.json`, `src/index.ts`, `.env` defaults.
- common/ — shared TypeScript package with `tsconfig.json`, `src/index.ts`, `src/versionInfo.ts`, builds to `dist/`.
