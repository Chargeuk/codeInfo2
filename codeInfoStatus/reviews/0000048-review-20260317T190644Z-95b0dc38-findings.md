## Findings

1. `should_fix` `generic_engineering_issue` The new conversation-summary client parser still normalizes malformed response fields into valid-looking success data, which can hide Story 48 contract drift around saved working-folder restore.
   - Evidence:
     - [`client/src/api/conversations.ts`](/Users/danielstapleton/Documents/dev/codeinfo2/codeInfo2/client/src/api/conversations.ts) `parseConversationSummary(...)` treats any non-`'MCP'` `source` as `'REST'`.
     - The same helper also converts any missing or non-object `flags` shape into `{}`, and an array would currently pass the loose `typeof record.flags === 'object'` check and be cast as a record.
     - Story 48 now depends on `flags.workingFolder` from the conversations API for restore/edit behavior, so a malformed server response can be silently flattened into “no working folder” instead of surfacing a contract violation.
   - Why this matters:
     - This branch intentionally tightened several request-side contracts to stop silent normalization of malformed data into success.
     - On the response side, the newly added client helper still hides malformed `source` and `flags` fields, which makes debugging server/client drift harder and can mask regressions in the exact persistence surface Story 48 introduced.
   - Contract comparison:
     - Before Story 48, the client did not depend on this new helper for working-folder restore semantics.
     - After Story 48, the route contract effectively treats `flags` as a meaningful persisted surface and `source` as structured metadata.
     - Defaulting invalid values into `'REST'` and `{}` is therefore backward-compatible in the narrowest sense, but it is not a truthful contract check for malformed modern responses.
   - Suggested review direction:
     - Keep backward compatibility for truly absent optional fields if needed, but reject or at least explicitly diagnose malformed present fields.
     - In particular, treat non-string present `source` values and non-plain-object present `flags` values as invalid response data rather than silently normalizing them into a valid-looking summary.
