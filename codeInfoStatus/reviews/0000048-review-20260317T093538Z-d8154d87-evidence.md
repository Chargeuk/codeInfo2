## Plan Extraction

### Description

Story 48 combines three coordinated behavior changes plus one scoped ingest bug fix:

- reference resolution moves from owner-first lookup to working-repo-first lookup across flows, direct command runs, and nested markdown resolution;
- working-folder selection becomes a persisted conversation contract across chat, agents, flows, and flow-created child agent conversations, with invalid saved folders cleared before reuse and with edits locked while runs are active;
- repo-owned environment variables move to the `CODEINFO_` and `VITE_CODEINFO_` namespaces in one clean cutover with no dual-read compatibility;
- OpenAI embedding token counting moves from the old heuristic to tokenizer-backed counting with the provider-truth 8192-token limit and explicit lifecycle handling.

The plan explicitly keeps the existing subfolder conventions, path-safety rules, and fail-fast policy for invalid higher-priority candidates. It also requires one shared repository-candidate-order contract, structured lookup logging, compact execution metadata, and deterministic stale-vs-operational working-folder handling.

### Acceptance Criteria

The plan defines 46 acceptance criteria. In summary, the branch must prove:

- one shared four-slot candidate order for every command and markdown lookup, restarted fresh at each nested hop;
- persisted and restorable working-folder state across all supported surfaces, with canonical absolute repository-root storage, idle edit support, in-flight lockouts, child-conversation inheritance, and stale-path clearing;
- explicit observability through structured lookup logs, compact runtime metadata, and documentation updates;
- a single env cutover to `CODEINFO_` and `VITE_CODEINFO_` across code, compose, wrappers, tests, and docs;
- tokenizer-backed OpenAI counting with 8192-token truth, deterministic oversized-input classification, no regression to the old heuristic, and explicit lifecycle handling.
