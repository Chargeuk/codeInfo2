## Canonical Plan Extract

### Description

Story 54 accelerates repository ingest for large prose-heavy text files by combining three changes in the existing ingest runtime path:

1. A large-text chunking path for large `.md`, `.mdx`, and `.txt` files.
2. Provider-aware embedding dispatch with bounded batching and bounded concurrency.
3. A conservative AST delta rule that skips AST work for non-AST-only deltas and otherwise reuses the existing full rebuild path.

The story explicitly keeps the existing ingest UI and existing ingest REST routes, does not add rename detection, does not add partial AST updates, and does not require a benchmark SLA.

### Acceptance Criteria

The plan defines 33 acceptance criteria covering discovery metadata, prose chunking behavior, batching and queue controls, cancellation semantics, AST delta rules, backward compatibility, documentation, and final validation.

### Out Of Scope

Key out-of-scope items in the plan:

- Partial changed-file AST updates.
- Rename detection for delta re-embed.
- New ingest API routes, browser surfaces, listeners, or worker services.
- Broad embedding-provider redesign outside ingest.
- Benchmark harnesses or hard performance thresholds.
- Unrelated performance work outside ingest.
