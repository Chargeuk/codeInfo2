## Plan Extract

### Description

Story 47 unifies Codex chat/runtime behavior around one shared configuration path. The intended outcome is that `codex/chat/config.toml` becomes the live default-model source for Codex chat surfaces, `Codex_model_list` order is preserved while unioning in the chat-config model, missing `codex/config.toml` and `codex/chat/config.toml` are bootstrapped from canonical in-code templates, and Context7 API key handling moves to an in-memory `CODEINFO_CONTEXT7_API_KEY` overlay rather than on-disk rewriting.

The plan is explicitly narrow about failure behavior. Existing invalid or unreadable config files must remain in place; bootstrap applies only to missing files. The plan also explicitly limits Context7 normalization to the local stdio `command` plus `args` shape already used in this repo, and it explicitly requires shared base-runtime inheritance so minimal chat bootstrap does not strip required execution settings.

### Acceptance Criteria

The plan defines 36 acceptance criteria covering:

- shared Codex model-list and default-model resolution across REST and MCP surfaces;
- deterministic missing-file bootstrap from in-code templates;
- preserved shared runtime inheritance after direct chat bootstrap;
- Context7 placeholder/no-key/explicit-key normalization without disk rewrites;
- unchanged public REST and MCP payload shapes.

### Out Of Scope

The plan keeps these areas out of scope:

- prompt, MCP tool-description, and workflow-step behavior changes;
- auth propagation changes for agent homes;
- rewriting existing user-edited config files during runtime;
- broader provider-selection changes beyond the stated model/default fixes;
- remote/header-based Context7 normalization;
- generic credential validation or a general config-migration framework.
