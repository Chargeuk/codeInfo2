## Active Plan Extract

### Description

Story 49 addresses long-transcript client rendering slowdowns across Chat, Agents, and Flows, with the most user-visible symptom being input lag in the Agents `Instruction` field when a long transcript is visible. The planned fix is a shared client-side transcript rendering path, isolation of page-local controls from transcript rerenders, virtualization for long transcripts, shared scroll and expansion-state ownership, and one narrow allowed server-side exception for stop-near-complete status alignment and diagnostics.

### Acceptance Criteria

The plan requires:

1. responsive long-transcript behavior, especially while typing on Agents;
2. no full rich-transcript rerender on every Agents keystroke;
3. one reusable shared transcript rendering path across Chat, Agents, and Flows;
4. removal of page-local inline transcript bubble loops from the three pages;
5. preserved newest-last display order;
6. virtualization or equivalent render windowing for long lists;
7. preserved rich transcript features, page-configured citations, hydration and inflight behavior, retained-assistant behavior on Flows, scroll anchoring, dynamic remeasurement, and expansion-state persistence;
8. shared ownership of transcript scroll and rich-row state;
9. stable transcript test targets after the refactor;
10. an explicit long-transcript validation path across Chat, Agents, and Flows;
11. no server API, websocket, or persistence-contract expansion beyond the narrow Task 3 stop-alignment and stop-path diagnostic exception;
12. a client structure that is easier to tune later because transcript logic is centralized.

### Out Of Scope

The plan explicitly excludes:

- general server-side API, websocket, or persistence-model changes;
- transcript feature removal for performance;
- unrelated UI redesign;
- unrelated dependencies;
- moving provider, agent, or flow controls into the shared transcript;
- changing hydration or retained-assistant semantics beyond preserving them;
- forcing always-on auto-scroll;
- general server performance work;
- new server listeners, health endpoints, runtime-config injection paths, Compose services, deployment mappings, or host source bind mounts;
- new transcript payload or persisted storage shapes.
