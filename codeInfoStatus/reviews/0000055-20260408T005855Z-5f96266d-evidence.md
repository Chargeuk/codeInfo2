## Canonical Plan Summary

### Description

Story 55 adds one durable Mongo-backed queue for ingest and re-embed requests, keeps runtime execution single-flight, preserves blocking behavior for re-embed callers while queued work waits, exposes queued state through queue-aware write contracts plus the shared repository-list read contract, and keeps queued work visible and submittable in the ingest UI.

### Out Of Scope

- user removal or cancellation of queued-but-not-started requests
- priority or non-FIFO queue behavior
- more than one active ingest worker at once
- degraded execution when Mongo is unavailable
- a dedicated queued-work card outside the repository list
- queue-position fields on unrelated non-repository read surfaces
- exactly-once crash semantics
