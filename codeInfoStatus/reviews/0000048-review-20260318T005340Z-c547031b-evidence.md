## Plan Summary

### Description

Story 48 combines three substantive product changes plus the review-fix chain that closed them:

1. Reference resolution moved from owner-first to working-repo-first for flow-command, flow-markdown, command-markdown, and direct-command lookups, with a single repository-candidate contract, deterministic duplicate handling, per-hop restart of nested lookup order, structured lookup logging, and compact lookup metadata.
2. The selected working folder became a persisted, editable, and restorable conversation-level concept across chat, agents, flows, and direct agent commands, with invalid-path clearing, idle-only edits, lock semantics during active runs, and inheritance into flow-created child agent conversations.
3. Repo-owned environment variables were cut over to `CODEINFO_` / `VITE_CODEINFO_`, and OpenAI ingest token counting moved from heuristics to tokenizer-backed counting with deterministic oversized-input classification and no silent heuristic fallback in OpenAI-specific paths.

### Acceptance Criteria

The plan defines `AC01` through `AC48` covering:

- working-repo-first candidate ordering, duplicate handling, nested lookup restart behavior, and command/markdown/direct-command resolution semantics;
- persisted working-folder save, restore, edit, inheritance, invalidation, and lock behavior across chat, agents, flows, and direct commands;
- structured lookup logs and compact run metadata as the canonical debugging surface;
- complete env cutover to `CODEINFO_` / `VITE_CODEINFO_` with no lingering repo-owned legacy names in checked-in files;
- tokenizer-backed OpenAI counting, provider-truth 8192-token limits, deterministic overflow classification, and explicit tokenizer lifecycle handling.
