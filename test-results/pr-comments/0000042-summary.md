## Story 0000042 Summary

Root cause:
Shared client transcript state in `client/src/hooks/useChatStream.ts` could still be mutated by stale older-inflight websocket events during Flow-style idle streaming. Flow exposed the bug most clearly because later step inflights could start while earlier bubble refs were still live, causing visible text loss until reload hydration restored the persisted content.

What changed:
The primary fix stayed in the shared client streaming path. `useChatStream` now ignores stale older-inflight non-final events, keeps stale `user_turn` replays from rebinding the active assistant bubble, and preserves late older `turn_final` handling non-destructively. `useChatWs` still owns same-inflight lower-sequence filtering before events reach the shared hook. `FlowsPage` only gained the page-level `flows.page.live_transcript_retained` verification marker; the conditional Task 7 Flow-page safeguard was not needed and remained N/A.
