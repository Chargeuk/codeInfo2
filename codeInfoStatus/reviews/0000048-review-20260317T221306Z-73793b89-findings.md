- `server/src/agents/service.ts`
- `server/src/flows/markdownFileResolver.ts`

- `server/src/agents/service.ts` emits the same marker with the same reduced schema.
- `server/src/flows/markdownFileResolver.ts` emits the same marker but adds `referenceType: 'markdownFile'`.

Why this matters:

- Story 48 explicitly treats structured lookup logs as the canonical debugging surface for reference resolution.
- The findings pass was required to compare shared log marker schemas across emitters and flag mismatches that make downstream parsing or operational debugging ambiguous.
- A consumer that keys off `DEV_0000048_T1_REPOSITORY_CANDIDATE_ORDER` now has to branch on emitter-specific optional fields instead of trusting a stable shape.

Why this is `should_fix`:

- The mismatch is localized to files already changed by Story 48.
- Fixing it should be low risk: make the marker schema uniform across emitters, likely by always including the same nullable envelope fields for `referenceType` and any other shared lookup-order metadata.
- This improves a shared contract without broadening the story or changing public API payloads.
